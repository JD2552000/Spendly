Slash Commands:
-> Slash Commands are shortcuts you type inside Claude Code session,
starting with '/', that triggers a specific predefined action or
worflow instantly - without writing out a full prompt.

-> These are built-in commands that ship with Claude Code and there
are Custom Commands that we build ourselves.

Sessions in Claude Code :
-> A session is a conversation with Claude-Code- everything from when
you run claude the first time to when you type /exit slash command.

-> It has a unique ID, captures full message history, all files reads,
all tool results, and current working directory.

-> Sessions are saved automatically to ~/.claude/projects/ and can be
resumed at any time, even after closing the terminal.
"claude -r" and then select the conversation you want to continue.

-> To go from one session to another session the slash command that can
be used is "/resume"

-> One session = one task.For example in a big application we can implement
one new feature as one new session. Rename your new session immediately.
Perform Commits frequenty within a session.

-> Use "/btw" for quick questions that is while building you had some doubt
which is out lets say of same topic but not directly related to what you are
building then use "/btw". Export a session before a big refactor of code so
that it creates a file in your directory that has all the conversational context
of this session which can be used in future.

/logout - Logout from Anthropic account
/login - Login to Anthropic account
/model - to switch between the model
/stats - On which day you used what model how many token and many information about
our Claude usage.
/insights - Give us insights about how we used Claude and what is our patter
and it recommends us to how to use Claude and how can we improve our usage.
/configure - Change our Claude Settings
/permissions - It is used to give whatever and how much access we want to give to 
different tools that Claude use. For example only read access to some tools and 
both read and write access to some tools depending on the usage.
/voice - allow voice assistant

-> Now whenever we want to tell Claude that we want to change the code in particular
file then we can use '@[file_path]'.This is called mentioning.
